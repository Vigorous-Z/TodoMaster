## 错误全记录（2个关键错误）

### 误1：`doLogout` 调用 `api_unbind_user_tasks` 破坏数据 ❌

- **原设计**：登出时调用 `api_unbind_user_tasks(owner_id)` 将用户任务归零为游客
- **问题**：再登录时所有任务的 `user_id` 都被设为了 NULL，`api_get_tasks({owner_id: uid})` 查不到任何任务
- **现象**：注册后列表空；登出再登录原账号，任务消失且无法恢复
- **修正**：登出只切视图 `loadTasks()`，不修改任何任务的归属字段

### 误2：`addTask` 参数顺序混乱导致 description 写入 project ❌

- **原设计**：`store.addTask(title, due, priority, tags, project)`，第5形参叫 `project`
- **调用**：`store.addTask(title, due, priority, [...tags], description.value)`——第5实参实际上是 `description`
- **问题**：description 的内容被写入 DB 的 project 字段，而 description 字段永远为空
- **修正**：`store.addTask(title, due, priority, tags, description, project)`，第5参数改为 description，第6为 project

## 动画清单（3个）

| 组件 | 动画类型 | 技术 | 参数 |
|------|---------|------|------|
| Settings | 左侧滑入/滑出 | `transform: translateX` + `transition` + CSS class 切换 | 0.6s `cubic-bezier(0.16, 1, 0.3, 1)` |
| addTask 面板 | 下拉/收起 | `max-height: 0 ↔ 400px` + `overflow: hidden` | 0.5s `cubic-bezier(0.16, 1, 0.3, 1)` |
| DetailPanel | 淡入淡出 | `<transition>` 无 mode + `position: absolute` 重叠 | 0.3s `ease` opacity |

### Settings 动画细节

- Settings 组件根元素 `position: absolute` 固定于侧栏区域
- 默认 `transform: translateX(-100%)` 隐藏于左侧屏外，`pointer-events: none`
- `.settings-visible` class → `translateX(0)`
- **重点**：不在 scoped 内定义 `.settings-visible`，用非 scoped `<style>` 块匹配外部传入 class

### 创建任务面板动画细节

- 原 `v-if` 改为始终渲染，用 `panel-wrapper` 包裹，通过 `max-height` 过渡
- 优点：高度平滑变化不影响下方任务列表 DOM 定位

### 详情面板动画细节

- `.detail-empty` 和 `.detail-content` 均为 `position: absolute`，重叠于同一区域
- `<transition name="detail-fade">` 去掉 `mode="out-in"`，实现旧元素淡出与新元素淡入同时进行

## 后续注意事项

1. **Settings 动画用 CSS class 切换，不用 v-if/v-show**——否则 DOM 销毁无法 transition
2. **scoped 样式无法匹配外部传入 class**——必须加非 scoped `<style>` 块
3. **创建任务参数顺序**：`(title, due, priority, tags, description, project)`，不要搞混
4. **登出不要修改任何任务数据归属**——只切换视图即可
