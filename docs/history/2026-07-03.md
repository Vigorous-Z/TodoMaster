## 遇到的坑与解决方案

### 坑1：PyWebView file:// 协议下 ESM 模块无法加载
- **现象**：`file:///.../index.html` 引入 ESM，CORS 阻止，窗口空白
- **解决**：Python 内嵌 `http.server` serve dist，PyWebView 加载 `http://127.0.0.1:PORT`

### 坑2：PyWebView js_api 是同步的，bridgeAdapter 误用 async/await
- **现象**：前端调用 bridge 返回 undefined
- **解决**：bridgeAdapter 全部改为同步调用

### 坑3：PyWebView 注入 window.pywebview.api 晚于 Vue onMounted
- **现象**：loadTasks() 时 api 未就绪，走了 localStorage，DB 有数据但前端看不到
- **解决**：App.vue 检测 `window.pywebview.api.api_get_tasks`，未就绪则监听 `pywebviewready` + 500ms 兜底

### 坑4：日期格式不一致
- **现象**：DB 存 ISO 8601（`2026-07-02T15:00:00`），前端需 `YYYY-MM-DD HH:MM`
- **解决**：to_frontend() 加 `_format_due()` 转换

### 坑5：JS 端 mock 数据残留
- **现象**：前端始终显示 8 条旧 mock，操作不写 DB
- **解决**：删除 JS 端 defaultTasks，种子数据移至 SQLite init_db()；id 统一为 UUID

## 当前架构

```
用户操作 → Vue 组件 → Pinia Store
                          ↓
                  getAdapter() 自动选择
                 /                    \
    bridgeAdapter（同步）     localAdapter（Promise）
    window.pywebview.api      localStorage
              ↓
    backend/api/handlers.py
              ↓
    backend/services/task_service.py
              ↓
    backend/repositories/local_task_repo.py
              ↓
    SQLite（data/data.db）
```

## 后续开发注意事项

1. **所有新增 PyWebView API 函数必须是同步的**（js_api 不支持 await）
2. **前端新增操作前必须先确保 bridge 就绪**（`isBridgeReady()` 检查）
3. **数据字段命名**：Python snake_case → 前端 camelCase，`to_frontend()` 做映射
4. **前端修改后必须重新 `npm run build`** 再启 PyWebView 才能看到效果
5. **PowerShell 不支持 `&&`**，用 `;` 分隔命令
6. **开发准则**：能跑的不动、要动的想好、不确定先问
