# TodoMaster 开发进程规划文档

## 1. 总体开发顺序

```
Phase 1: 基础模块
    ↓
Phase 2: 用户模块
    ↓
Phase 3: 云端模块（数据同步优先，AI 代理后续）
    ↓
Phase 4: AI 模块
```

> **并行潜力**：Phase 3 云端模块中的“AI 代理服务”和 Phase 4 可合并为一次交付，同步服务独立上线。

> **远期规划**：进阶模块（重复任务、排序、标签、统计报表等）作为独立的可插拔扩展，在基础模块稳定后按需迭代，不阻塞主线交付。


## 2. Phase 1：基础模块

### 目标
**“任务增删改查完全本地化”** — 前端不再使用 localStorage，所有数据读写走 PyWebView + Python 后端 + SQLite。

### 子任务清单

| # | 子任务 | 依赖 | 产出 |
|---|--------|------|------|
| 1.1 | 建立后端目录结构 | 无 | `backend/` 完整骨架 |
| 1.2 | SQLite 初始化与连接池 | 1.1 | `core/database.py`，首次启动自动建表 |
| 1.3 | Task 数据模型定义 | 1.1 | `models/task.py`（dataclass/Pydantic） |
| 1.4 | Task 仓储层实现 | 1.2, 1.3 | `repositories/local_task_repo.py`（CRUD） |
| 1.5 | Task 服务层实现 | 1.4 | `services/task_service.py`（业务逻辑） |
| 1.6 | PyWebView 桥接接口 | 1.5 | `api/handlers.py`（`api_*` 函数） |
| 1.7 | 前端 Pinia Store 改造 | 1.6 | `stores/taskStore.js`（调用后端而非 localStorage） |
| 1.8 | 基础功能回归测试 | 1.7 | 所有任务操作（增删改查/切换视图/详情）正常 |

### 完成标准
- 前端启动后，任务数据从 SQLite 读取。
- 新增/修改/删除任务后，数据持久化到 SQLite。
- 关闭应用再打开，数据不丢失。
- localStorage 中不再存储任务数据。

### 不包含（留待后续）
- 重复任务
- 排序与标签系统
- 统计报表


## 3. Phase 2：用户模块

### 目标
**“游客模式 + 注册/登录本地状态管理”** — 用户可以在本地完成注册和登录，Token 保存在本地配置中，云端认证接口使用 Mock（Phase 3 再打通）。

### 子任务清单

| # | 子任务 | 依赖 | 产出 |
|---|--------|------|------|
| 2.1 | User 数据模型定义 | 1.1 | `models/user.py`（本地缓存） |
| 2.2 | 配置读写模块 | 1.1 | `core/config.py`（Token/偏好存储） |
| 2.3 | 用户服务层（本地状态） | 2.1, 2.2 | `services/user_service.py`（游客/登录状态） |
| 2.4 | 注册/登录界面（前端） | 2.3 | 设置页中的注册/登录表单 |
| 2.5 | Mock 认证接口（后端） | 2.3 | 本地模拟注册/登录，返回假 Token |
| 2.6 | 用户状态 Pinia Store | 2.4, 2.5 | `stores/userStore.js` |
| 2.7 | 游客模式验证 | 2.6 | 未登录状态所有功能可用，同步入口灰显 |

### 完成标准
- 用户打开应用，自动进入游客模式。
- 用户可在设置页注册“账号#数字”并设置密码。
- 注册后可用账号+密码登录，本地记住 Token。
- 登出后回到游客模式，本地数据保留。
- 云端认证尚未真正打通（Mock 已就绪，为 Phase 3 预留接口）。

### 不包含
- 云端真实注册/登录（Phase 3）
- 密码找回邮件（后续迭代）
- 云端用户数据同步（Phase 3）


## 4. Phase 3：云端模块

### 目标
**“数据同步优先，AI 代理后续”** — 先解决多端数据同步的核心问题，AI 代理作为独立服务在后期交付。

### 子任务清单

| # | 子任务 | 依赖 | 产出 |
|---|--------|------|------|
| 3.1 | 云端 FastAPI 骨架 | 无 | `cloud/app.py`，路由结构 |
| 3.2 | PostgreSQL 数据库初始化 | 3.1 | `cloud/db/`，用户表 + 任务表 + 聊天表 |
| 3.3 | 用户注册/登录 API（真实） | 3.2 | `/auth/register`，`/auth/login`（JWT） |
| 3.4 | 客户端对接真实认证 | 3.3, 2.6 | 前端注册/登录走真实云端 API |
| 3.5 | 同步 diff API | 3.2 | `/sync/diff`（返回差异和冲突列表） |
| 3.6 | 同步 apply API | 3.5 | `/sync/apply`（执行合并） |
| 3.7 | 客户端同步协调器 | 3.5, 3.6 | `services/sync_service.py`（三步同步流程） |
| 3.8 | 冲突解决 UI（前端） | 3.7 | 冲突弹窗 + 逐条选择 |
| 3.9 | AI 代理服务部署 | 3.1 | `/ai/parse`（转发请求，持官方 Key） |

### 完成标准
- 用户注册/登录真实走云端 PostgreSQL。
- 登录后，点击“同步”可将本地任务上传至云端。
- 另一设备登录同一账号，可拉取云端数据到本地。
- 两端同时修改同一任务时，触发冲突弹窗，用户选择保留哪一方。
- AI 代理服务可用，可通过 `api.ai_parse()` 调用（虽然 AI 模块尚未开发，但接口已就绪）。

### 说明
- 子任务 3.1–3.8 是数据同步核心，**优先交付**。
- 子任务 3.9（AI 代理）可与 Phase 4 AI 模块合并交付，提前部署也可。


## 5. Phase 4：AI 模块

### 目标
**“AI 作为日程管理的辅助工具”** — 支持自然语言解析、任务总结、效率建议、聊天对话。

### 子任务清单

| # | 子任务 | 依赖 | 产出 |
|---|--------|------|------|
| 4.1 | AI 输入框模式切换（前端） | 1.7 | “添加任务” ↔ “AI 解析”切换按钮 |
| 4.2 | AI 解析 API 对接（前端→后端） | 4.1, 3.9 | 输入自然语言 → 创建结构化任务 |
| 4.3 | AI 助手标签页（右侧栏） | 1.7 | `DetailPanel.vue` 新增“AI 助手”标签 |
| 4.4 | ChatSession 数据模型 + 仓储 | 1.2, 1.3 | `models/chat.py` + `repositories/local_chat_repo.py` |
| 4.5 | 聊天对话 UI | 4.3, 4.4 | 对话展示区 + 输入框 + 流式输出 |
| 4.6 | 聊天历史列表 | 4.5 | 右侧栏覆盖式历史列表，收藏置顶 |
| 4.7 | AI 快捷指令（/总结今天 等） | 4.2, 4.5 | 输入框快捷按钮 + 系统 Prompt 注入任务上下文 |
| 4.8 | AI 模块全面测试 | 4.7 | 解析、总结、建议、对话均正常 |

### 完成标准
- 用户可通过输入框模式切换，用自然语言创建任务。
- 用户可进入 AI 助手标签页进行多轮对话。
- AI 能根据当前任务数据生成“今日总结”“本周报告”等摘要。
- 对话历史保存在本地 SQLite，可查看、恢复、收藏。
- 收藏对话在历史列表中置顶。

### 不包含
- 聊天历史云端同步（待 Phase 3 同步模块扩展）
- AI 模型切换/配置（后续迭代）


## 6. 里程碑总览

| 里程碑 | 包含阶段 | 交付物 |
| :--- | :--- | :--- |
| **M1：本地基石** | Phase 1 | 完全本地化的任务管理应用，SQLite 替代 localStorage |
| **M2：用户就绪** | Phase 2 | 游客/登录状态管理，注册登录界面，本地 Token 存储 |
| **M3：云端打通** | Phase 3 | 多端数据同步可用，冲突解决机制，AI 代理服务就绪 |
| **M4：AI 赋能** | Phase 4 | AI 助手完整功能，任务解析 + 总结 + 聊天 |


## 7. 各阶段启动条件

| 阶段 | 前置条件 | 可提前准备 |
| :--- | :--- | :--- |
| Phase 1 基础模块 | 架构文档已确认 | ✅ |
| Phase 2 用户模块 | Phase 1 完成 | ✅ |
| Phase 3 云端模块 | Phase 2 完成（认证接口依赖） | 可先部署云端骨架，并行开发 |
| Phase 4 AI 模块 | Phase 3 AI 代理就绪 | 可先在本地用 mock 开发，后续切换真实代理 |


## 8. 并行化建议（提速）

| 并行组合 | 说明 |
| :--- | :--- |
| Phase 1 + 云端 AI 代理（3.9） | 基础模块开发的同时，可先部署 AI 代理服务，不阻塞后续 AI 模块进度 |
| Phase 2 + 云端数据同步（3.1–3.4） | 用户模块本地开发的同时，可搭建云端认证服务，Phase 2 完成后直接切换真实接口 |
| Phase 3 + 云端 AI 代理 | 同步服务开发不受影响，AI 代理可并行完成 |


## 9. 验收方式

每个 Phase 完成后，执行以下验收：

| 阶段 | 验收方式 |
| :--- | :--- |
| Phase 1 | 手动测试所有任务操作，确认数据持久化到 SQLite |
| Phase 2 | 注册/登录流程走通，Token 存储正常，登出不影响本地数据 |
| Phase 3 | 两台设备登录同一账号，同步后数据一致；修改同一任务，冲突弹窗出现 |
| Phase 4 | 自然语言创建任务、总结今日任务、多轮对话均正常 |
