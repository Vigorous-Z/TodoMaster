## 错误全记录（4个关键错误）

### 错1：设置页做成弹窗 ❌
- **原设计**：`position: fixed` 全屏遮罩
- **用户反馈**："左侧栏拉出设置页面替换掉原页面"——就是字面意思
- **教训**：用户描述就是实现规格，不要往上加自己的想法

### 错2：userStore.init() 缓存 bridge 状态 ❌
- **原设计**：`init()` 时 `this._useBridge = isBridgeReady()`，之后永远用缓存值
- **问题**：`init()` 在 `pywebviewready` 之前执行，`_useBridge = false` 永不会变
- **教训**：每次 action 调用时实时检测 bridge，永远不要缓存

### 错3：userStore 没有 pywebviewready 等待 ❌（最致命）
- **原设计**：`userStore.init()` 直接在 `onMounted` 里调用
- **问题**：此时 PyWebView API 未就绪，永远走 `localAdapter` 写 localStorage
- **现象**：注册显示成功→DB 无记录→重启后消失
- **修正**：`loadUser()` 与 `loadTasks()` 合并为 `loadAll()`，统一在 `pywebviewready` 后执行
- **教训**：所有需要 bridge 的 init 操作必须在 `pywebviewready` 事件或 500ms 兜底之后执行

### 错4：修复时继续犯同样错误 ❌
- 第一次修复只去掉了 `_useBridge` 缓存，但 `loadUser()` 仍然直接调用，没有等待 `pywebviewready`
- 第二次修复加了实时检测，Settings.vue 改为 async/await，但 `loadUser()` 仍然没等
- 第三次才把 `loadUser()` 与 `loadTasks()` 合并进 `pywebviewready` 等待逻辑
- **教训**：修 bug 必须从根因修，不是从一个表象跳到另一个表象

## 固化的前后端联通模式（必须严格遵守）

### 启动时序
```
App.vue onMounted()
    ↓
检查 window.pywebview.api 是否就绪?
    ├── 是 → 立即 loadAll()
    └── 否 → 监听 pywebviewready 事件
              + setTimeout(500ms) 兜底
              → loadAll()
```

### Store 双适配器模式
```js
// 每次 action 实时检测
function getAdapter() {
    if (isBridgeReady()) return bridgeAdapter
    return localAdapter
}

async register(prefix, password) {
    const adapter = getAdapter()  // ← 必须每次实时
    const result = await adapter.register(prefix, password)
    // ...
}
```

### 后端三层
```
handlers.py:  def api_xxx(data: dict) → 接收单一 dict 参数 → return service层
services:     纯函数或类 → 业务逻辑 → return dict
database.py:  get_connection() → conn.execute → 每次新建连接用完关闭
```

### ApiBridge 注册
```py
class ApiBridge:
    api_register = staticmethod(api_register)
    # 全部是 staticmethod + 同步函数
```

### 重要约束
- Python `def` 同步函数，不能 `async def`
- 前端 bridgeAdapter 同步调用，无 `await`
- `api_*` 函数一律接收 `data: dict`
- **新增 Store init 操作必须等待 pywebviewready**

## 后续开发铁律

1. **每个新模块的 init 操作必须进入 `pywebviewready` 等待机制**
2. **每次都实时 `getAdapter()`，禁止缓存 bridge 状态**
3. **前端改动后必须 `npm run build`**
4. **用户描述就是实现规格，不擅自翻译**
5. **新增 API：Python → ApiBridge → bridgeAdapter 三处同步改**
6. **修 bug 必须找到真正的根因，不要修表面现象**
