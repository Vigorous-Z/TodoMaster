# TodoMaster 架构设计文档 v1.0

## 1. 项目定位

**TodoMaster** 是一款 **本地优先（Local-First）** 的日程管理桌面应用。

- **核心原则**：离线可用，数据在本地，云端仅为同步和 AI 代理服务。
- **用户形态**：支持游客模式（纯本地）和注册用户（跨设备同步）。
- **AI 定位**：日程管理的辅助工具，承担任务解析、总结、建议三大职责，不喧宾夺主。


## 2. 总体架构

### 2.1 三层架构

| 层级 | 组件 | 部署位置 | 职责 |
| :--- | :--- | :--- | :--- |
| **客户端** | Vue 3 前端 + Python 本地后端 | 用户电脑（PyWebView 打包） | UI 渲染、本地 CRUD、AI 路由决策、同步协调 |
| **云端网关** | Python 代理服务 | 云服务器 | AI 代理（持有官方 Key）、用户鉴权、同步 API |
| **云端存储** | PostgreSQL | 云服务器 | 用户账号、云端任务数据、聊天历史 |

### 2.2 数据流

```
用户操作 → Vue 组件 → Pinia Store → PyWebView API 桥接 → Python 本地后端
                                                              ↓
                                                    SQLite（本地数据）
                                                              ↓
                                                    云端同步（按需触发）
                                                              ↓
                                                    PostgreSQL（云端数据）
```


## 3. 技术栈

### 3.1 前端（Vue 3）

| 类别 | 技术 | 说明 |
| :--- | :--- | :--- |
| 框架 | Vue 3 | 组合式 API，`<script setup>` 语法 |
| 构建 | Vite | 极速开发与构建 |
| 状态管理 | Pinia | 全局状态（任务、视图、会话） |
| 语言 | JavaScript | 纯 JS，不引入 TS（降低 AI 编码门槛） |
| 样式 | 原生 CSS | CSS 变量 + Scoped 隔离 |

### 3.2 本地后端（Python）

| 类别 | 技术 | 说明 |
| :--- | :--- | :--- |
| WebView 容器 | PyWebView | 加载 Vue 构建产物，暴露 js_api 桥接 |
| 数据库 | SQLite | 本地数据存储（用户目录下） |
| 配置存储 | JSON 文件 | API Key、用户偏好（`~/.todomaster/config.json`） |
| HTTP 客户端 | requests | 调用云端 AI 代理和同步 API |

### 3.3 云端后端（Python）

| 类别 | 技术 | 说明 |
| :--- | :--- | :--- |
| Web 框架 | FastAPI | 轻量、异步、自动生成 API 文档 |
| 数据库 | PostgreSQL | 用户数据、云端任务、聊天历史 |
| 认证 | JWT | 用户登录态管理 |
| 密码哈希 | bcrypt | 用户密码存储 |
| AI 代理 | requests + 流式响应 | 转发请求到大模型 API |

## 7. 核心业务逻辑

### 7.1 任务 CRUD

- 所有写操作（增/改/删）必须更新 `updated_at` 为当前时间。
- 删除使用软删除（设置 `deleted_at`），不物理删除数据。
- `status` 为 `'completed'` 的任务视为已完成，不参与“今日任务”统计。

### 7.2 重复任务（存储在 `extras.recurrence`）

**数据结构**：

```json
{
  "recurrence": {
    "frequency": "daily" | "weekly" | "monthly" | "yearly",
    "interval": 1,
    "weekdays": [1, 3, 5],
    "month_date": 15,
    "start_date": "2026-07-10",
    "end_date": "2026-12-31",
    "count": 10,
    "exceptions": ["2026-07-17"]
  }
}
```

**核心算法**：

- **不预生成实例**：通过 `get_next_occurrence()` 实时计算。
- **覆盖实例（Override）**：用户修改单次日程时，创建新任务，`parent_uuid` 指向母版，`extras.is_override = true`。
- **前端展示**：调用 `generate_occurrences(start_date, end_date)` 获取时间窗口内的所有实例。

### 7.3 AI 模块

**输入框模式切换**：

- 单输入框 + 模式切换按钮（📝 添加任务 / ✨ AI 解析）。
- AI 不可用时，切换按钮禁用，灰显提示。

**AI 能力**：

| 能力 | 触发方式 | 实现 |
| :--- | :--- | :--- |
| 任务解析 | 用户输入自然语言 | System Prompt 引导输出结构化 JSON |
| 任务总结 | 用户输入“/总结今天”等快捷指令 | AI 读取本地任务数据后生成摘要 |
| 效率建议 | 用户自由提问 | AI 基于任务上下文给出建议 |

**AI 调用路由**：

```
用户输入 → ai_service.parse()
              ↓
      检查是否配置了自定义 Key
              ↓
    有自定义 Key → 直接调用用户指定的大模型 API
    无自定义 Key → 调用云端代理服务（持官方 Key）
```

**降级策略**：

- AI 服务不可用时，AI 输入框灰显，提示“AI 服务暂时不可用，请使用快速添加”。
- 不影响任务添加主流程。

### 7.4 同步策略

**冲突检测**：比较本地和云端的 `updated_at`，若两者都在 `last_sync_time` 之后修改过，视为冲突。

**同步流程（三步走）**：

1. **客户端发起 `/sync/diff`**：携带本地所有任务（含 `uuid`、`updated_at`）和 `last_sync_time`。
2. **云端返回差异列表**：`{ to_pull, to_push, conflicts }`。
3. **客户端处理冲突后，提交 `/sync/apply`**：包含用户的选择决议。

**冲突解决**：

- 弹出冲突窗口，逐条展示本地版本 vs 云端版本（含 `updated_at`、标题、内容）。
- 用户选择“保留本地”或“使用云端”。
- 支持“稍后处理”，冲突任务标记为 `sync_status = 'conflict'`，在同步状态栏提示。

### 7.5 用户体系

**账号格式**：`用户名#五位数字`（如 `张三#12345`），`#` 为固定分隔符。

**注册流程**：

1. 用户输入自定义前缀（如 `张三`）。
2. 后端检查是否存在相同前缀，分配一个未使用的 5 位随机数字。
3. 展示完整 ID 给用户确认（可“换一个数字”）。
4. 用户设置密码（可选绑定邮箱）→ 注册完成。

**登录**：支持“完整 ID + 密码”或“邮箱 + 密码”两种方式。

**找回密码**：

- 用户输入绑定的邮箱 → 云端发送重置链接（含 15 分钟有效 token）。
- 用户点击链接重置密码。

**游客模式**：

- 未登录状态下，应用所有功能可用（增删改查、AI 解析）。
- 数据仅存储在本地 SQLite，不触发同步。
- 设置页显示“登录以启用云同步”入口。


## 8. 降级与容错

| 场景 | 降级策略 |
| :--- | :--- |
| **AI 服务不可用** | AI 输入框灰显，提示“AI 服务暂时不可用”，快速添加通道保持可用 |
| **云端同步服务不可用** | 本地操作完全正常，同步按钮显示“同步失败，点击重试” |
| **Token 过期** | 自动刷新 Token；刷新失败则提示用户重新登录 |
| **本地 SQLite 损坏** | 启动时检测到损坏，提示用户恢复备份（`data.backup_*.db`） |
| **数据库版本不匹配** | 应用启动时自动执行迁移脚本；迁移失败则提示用户备份后重试 |

## 11. 开发优先级（交付顺序）

| 阶段 | 交付内容 | 说明 |
| :--- | :--- | :--- |
| **Phase 1** | 本地后端（SQLite + CRUD + 配置） | 替换 Pinia localStorage，接入 Python 后端 |
| **Phase 2** | AI 解析（输入框 + 解析 API） | 输入框模式切换，AI 解析任务并创建 |
| **Phase 3** | AI 聊天（右侧栏多标签页 + 对话） | 聊天会话、历史记录、收藏 |

| **Phase 5** | 数据同步（diff + apply + 冲突解决） | 三步同步流程、冲突 UI |
| **Phase 6** | 云端部署（FastAPI + PostgreSQL） | 云端服务上线，正式支持多设备 |
