# 智能日程管理助手 —— 项目简介、功能介绍与技术路线（最终版）

## 一、项目简介

### 1.1 项目背景

传统日程管理工具（如待办清单、日历应用）普遍存在**录入成本高**的问题——用户需要手动拆解任务、设定截止时间、划分优先级，这个过程本身消耗了大量认知资源。与此同时，大语言模型在自然语言理解与信息抽取方面展现出了卓越能力。本项目将两者结合，致力于打造一款 **“想说就能记，记完就能用”** 的智能日程助手。

### 1.2 项目定位

基于大语言模型的 **AI 原生日程管理工具**。用户使用口语化自然语言描述日程（如 *“后天下午三点提醒我给导师发邮件，顺便查一下文献”*），系统通过大模型自动解析为结构化日程信息，**在格式校验通过后直接自动录入数据库**，全程无需手动填表、无需逐项确认——真正实现“用嘴写日程”。

同时，系统提供灵活的**自动化策略配置**：用户可通过设置开启“二次确认”模式，让 AI 解析后先展示解析结果，经人工审核无误后再录入，满足对准确性要求更高的场景。

### 1.3 核心特色

- **自然语言 → 结构化日程**：用“说人话”代替手动填表，录入效率提升 10 倍以上

- **自动校验 + 智能重试**：自主实现 AI 输出格式校验，异常时自动反馈错误并让大模型修正，无需人工干预

- **默认自动录入，确认可配置**：AI 解析合格后默认直接入库，也可按需开启二次确认，兼顾效率与准确性

- **语音输入可选增强**：集成语音识别能力，支持“说出来就能记”，作为可选组件按需启用

## 二、功能介绍

### 2.1 核心功能（必选）

#### （1）智能日程解析

用户输入自然语言文本（或通过语音转文字后输入），系统调用大模型 API 进行语义解析，自动提取出标题、截止时间、优先级、标签等结构化日程信息。

| 用户输入（口语化）              | AI 解析输出（结构化 JSON）                                                                                              |
| ---------------------- | -------------------------------------------------------------------------------------------------------------- |
| “后天下午三点提醒我给导师发邮件”      | `{"title":"给导师发邮件","due_date":"2026-07-04 15:00:00","priority":"high"}`                                        |
| “下周一早上9点到11点开会讨论项目进度”  | `{"title":"项目进度会议","due_date":"2026-07-06 09:00:00","duration":120,"priority":"high"}`                         |
| “周五之前完成季度报告，顺便约一下财务对账” | `{"title":"完成季度报告","due_date":"2026-07-03 23:59:59","priority":"high"}, {"title":"约财务对账","priority":"medium"}` |

#### （2）格式自动校验与智能重试（自主实现）

- 系统对 AI 返回的 JSON 进行**严格格式校验**（字段完整性、类型正确性、日期合法性、优先级取值范围等）

- 校验失败时，系统自动将**具体错误信息**（如“缺少 title 字段”或“due_date 格式非法”）反馈给大模型，要求其重新生成（最多重试 3 次）

- 全部失败后降级为手动输入模式，确保系统在任何情况下都不中断用户操作

#### （3）自动化插入策略（灵活可配）

AI 解析通过格式校验后，按以下策略处理：

| 模式           | 行为                            | 适用场景          |
| ------------ | ----------------------------- | ------------- |
| **自动模式（默认）** | 解析合格后**直接自动插入**数据库，用户无需任何额外操作 | 日常高频录入，追求极速效率 |
| **确认模式（可选）** | 解析合格后展示“确认卡片”，用户审核确认后方可插入     | 重要日程、容错要求高的场景 |

用户可通过**配置文件**（`config.yaml`）或**命令行参数**（`--confirm` / `--auto`）自由切换两种模式。

#### （4）常规日程管理

- 查看所有日程（支持按时间、优先级、标签筛选）

- 手动添加 / 编辑 / 删除日程

- 标记完成 / 未完成

- 数据持久化存储（SQLite 数据库）

### 3.3 核心模块详细设计

#### （1）AI 解析与格式校验模块（AIService）

**设计原则**：不依赖 Instructor 等强制结构化库，完全自主控制解析、校验、重试全流程。

**核心流程（重点体现自动插入策略）**：

python

def process_user_input(user_text: str) -> Task:
    # 1. 调用大模型（首次）
    raw = call_llm(user_text)

    # 2. 格式校验 + 智能重试（最多3次）
    for attempt in range(3):
        try:
            data = json.loads(raw)
            validated_data = validate_schema(data)  # 自研校验
            break
        except (json.JSONDecodeError, ValidationError) as e:
            if attempt == 2:
                raise Exception("AI 解析失败，请手动输入")
            error_feedback = str(e)  # 如："缺少 title 字段"
            raw = call_llm(user_text, feedback=error_feedback)  # 带错误反馈重新调用
    
    # 3. 根据配置决定是否二次确认
    if config.get("confirm_mode", False):
        show_confirm_card(validated_data)
        if not user_confirms():
            return manual_input()
    
    # 4. 自动插入（默认行为）
    return task_service.add(validated_data)

**格式校验器（自研）** 包含以下检查项：

| 校验项   | 检查内容                          | 错误反馈示例                                 |
| ----- | ----------------------------- | -------------------------------------- |
| 字段完整性 | 是否包含 `title`、`due_date` 等必填字段 | "缺少 title 字段"                          |
| 类型正确性 | `title` 为字符串，`priority` 为枚举值等 | "priority 应为 'low'/'medium'/'high' 之一" |
| 日期合法性 | `due_date` 是否为有效时间格式          | "due_date 格式应为 YYYY-MM-DD HH:MM:SS"    |
| 时长合理性 | `duration` 为正整数               | "duration 应为正整数"                       |

### 3.5 关键技术挑战与应对

| 挑战               | 应对方案                              |
| ---------------- | --------------------------------- |
| AI 输出 JSON 格式错误  | 自研校验器 + 自动错误反馈重试（最多3次）            |
| 时间解析错误（如“后天”被误判） | Prompt 中明确当前日期，配合日期解析库做二次校验       |
| 自动插入导致用户“不放心”    | 提供 `confirm_mode` 配置项，用户可随时开启确认模式 |
| API 网络延迟         | 设置超时熔断（5秒），超时降级为手动输入              |
| 语音识别依赖网络         | 支持切换本地离线引擎（如 CMU Sphinx）作为备选      |

## 四、总结

本项目以 **“自然语言直达日程”** 为核心体验，通过大模型 API 将口语化描述一键转化为结构化日程，默认自动入库，真正实现“零摩擦”录入。自主实现的格式校验与智能重试机制确保了 AI 输出的稳定性与可靠性，而灵活的自动化策略配置（自动/确认模式切换）则兼顾了效率与可控性。

技术路线覆盖了 CLI 开发（Click + Rich）、AI 集成（DeepSeek + 自研校验）、数据持久化（SQLAlchemy + SQLite）、语音识别（可选）等多个 Python 核心方向，是一套**完整、可落地、具备极强扩展性**的 AI Native 应用实践。
